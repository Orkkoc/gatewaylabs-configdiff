# Contributing to ConfigDiff

Thanks for your interest in contributing.

This repository is the public companion repository for ConfigDiff, so contributions are most useful when they improve the public-facing experience around the product.

## Good Contribution Areas

- Documentation improvements
- Roadmap feedback
- Clarifications around use cases or terminology
- Sanitized examples or sample outputs
- Issue reports and reproducible bug descriptions
- Suggestions for future open-source tooling or policy packs

## Before You Start

Please open an issue before starting a larger change. That helps align scope early and avoids work landing in an area that is still private or intentionally out of scope for this repository.

For small typo fixes or straightforward documentation edits, a direct pull request is fine.

## Scope Rules

Please do not submit:

- Secrets, credentials, or internal environment details
- Customer data or real production configuration
- Proprietary code copied from private systems
- Unsanitized screenshots, logs, or diff outputs

## How to Contribute

1. Open or find an existing issue
2. Confirm the change fits this repository's public scope
3. Make the update in a focused branch
4. Submit a pull request with a clear description of the problem and fix

## Pull Request Notes

Good pull requests usually include:

- A concise summary of the change
- The reason the change is needed
- Any screenshots or sample output if the change affects documentation or examples
- A note about any follow-up work that still remains

## Communication

Please keep discussions constructive, specific, and respectful. Clear reports and narrowly scoped proposals are the fastest way to move things forward.
