# Roadmap

This roadmap describes the intended public direction for ConfigDiff. It is a planning document, not a release contract.

## Near Term

- Define the core comparison model for environment snapshots
- Publish clearer documentation for the public repository
- Add sanitized examples of drift reports and expected outputs
- Shape the open-source boundary for any shared utilities

## Next

- Introduce scheduled diff workflows
- Add alerting and notification concepts
- Define policy-based drift checks
- Document integration patterns at a high level

## Later

- Publish reusable policy packs or templates
- Release lightweight open-source helpers or CLI tooling
- Add more complete examples and onboarding flows
- Expand governance and audit-oriented reporting concepts

## Notes

Priorities may shift as the private product evolves and community feedback comes in.
